# WeekReport

## 1. Определение

**WeekReport** — еженедельный отчёт, генерируемый Стратегом (R1) на основе коммитов, DayPlans и WeekPlan. Фиксирует факт работы за неделю и служит входом для Session Prep (S02) и публикации итогов (R21 Публикатор).

## 2. Назначение

Без WeekReport нет ретроспективы: что реально сделано vs. что планировалось. Отчёт даёт: (1) количественные метрики (коммиты по репо), (2) качественные итоги (закрытые РП, достижения), (3) материал для публикации.

## 3. Структура

### 3.1. Frontmatter

```yaml
---
type: week-report
week: W{N}
date_start: YYYY-MM-DD
date_end: YYYY-MM-DD
status: draft | published
agent: Стратег
---
```

### 3.2. Секции

| Секция | Описание | Обязательная |
|--------|----------|-------------|
| Статистика по репо | Таблица: репо, коммиты, основная работа | Да |
| Закрытые РП | Список done-РП за неделю | Да |
| Ключевые достижения | 3-7 bullet points | Да |
| Completion rate | done / in_progress / pending | Да |
| Пост итогов | Markdown для публикации в клубе | Нет |

## 4. Жизненный цикл

```
Генерация (S03, Пн 00:00) → Ревью (R14) → Публикация (R21, Пн 06:00) → Вход для S02
```

## 5. Критерии качества

| Критерий | Проверка |
|----------|----------|
| Полнота | Все репо с коммитами учтены? |
| Точность | Статистика коммитов совпадает с git log? |
| Публикуемость | Пост итогов годен для клуба без редактуры? |
