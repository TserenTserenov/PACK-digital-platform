# [DP.SC.119] Рабочее пространство из браузера

## Обещание

**Кому:** Участник, который работает через браузер (claude.ai, ChatGPT, бот) и не использует VS Code / терминал.

**Зачем:** IWE не должно требовать технических навыков. 5 минут до первой пользы — не до первой настройки. Пользователь не знает что такое git — и ему не нужно.

**Что получит:**
- **Первая польза за 60 секунд** — ответ из базы знаний МИМ, который качественнее обычного ChatGPT
- **Классификация** — AI определяет новичок или знакомый с МИМ, адаптирует путь
- **Рабочее пространство** — репозиторий `DS-strategy` создан автоматически, пользователь видит «личная база знаний»
- **Знания в контексте** — поиск по платформенным (L2) и своим (L4) знаниям через один Gateway URL
- **Запись и хранение** — заметки, результаты диагностики, маршруты сохраняются через AI, без ручного git
- **Прогрессивное усложнение** — по мере роста AI предлагает структурировать, подключить новые возможности

**Что НЕ входит в обещание:**
- Протоколы ОРЗ (day-open, close) — требуют Claude Code
- Skills с файловыми операциями — требуют доступ к FS
- Hooks, pre-commit проверки — требуют локальную среду
- Memory-система (auto-memory) — требует Claude Code

**Критерий приёмки:** От первого сообщения до полезного ответа < 60 секунд. От первого сообщения до первой записи в личную базу < 10 минут. Пользователь ни разу не видит git CLI. Слова GitHub, repo, commit не произносятся до момента, когда пользователь сам спрашивает «а где хранится?».

## Ключевые принципы онбординга

1. **Ценность → действие → техника.** Сначала «зачем», потом «попробуй», техника под капотом
2. **Язык ценности, не техники.** «Личная база знаний», не «GitHub репозиторий»
3. **Один рекомендованный шаг.** Не 5 вариантов, а одно конкретное действие
4. **Подписка — не барьер.** Trial 30 дней, говорим о подписке когда возникает потребность, не на старте
5. **Экзоскелет, не автопилот (DP.D.046).** Усиливать мышление, не заменять

## Пользовательский путь

> Участник открывает AI-клиент с подключённым IWE, задаёт первый вопрос — AI ведёт его от первой пользы до рабочего пространства через понятный диалог.

| # | Шаг | Кто | Что происходит | Техническое |
|---|-----|-----|----------------|-------------|
| 0 | Определение контекста | AI | Классификация: новый / знакомый с МИМ / опытный | `get_instructions` + `dt_read_digital_twin` + `personal_list_sources` |
| 1 | Приветствие + ценность | AI | Объясняет зачем IWE на языке пользы. Один классифицирующий вопрос: «знаком с МИМ?» | Текст |
| 2 | Первое полезное действие | Участник | Задаёт вопрос / запускает Навигатора / Диагноста → получает ответ из базы знаний | `knowledge_search`, роли R27/R28 |
| 3 | Предложение сохранить | AI | «Хочешь, чтобы я запоминал?» — мягкое, после 1-2 полезных взаимодействий | Текст |
| 4 | Создание личной базы | Платформа | GitHub OAuth + `create_repository(name: "DS-strategy")`. Пользователь видит: «Личная база создана» | `github_connect` + `create_repository` |
| 5 | Первая запись | AI | Сохраняет результат предыдущего взаимодействия | `personal_write(source: "DS-strategy")` |
| 6 | Подписка | AI | Только при прямом вопросе или окончании trial. Две подписки: МИМ + ИИ | Текст |
| 7 | Прогрессивное усложнение | AI | 5+ заметок → «организуем?». 10+ → «структурируем в систему?». Регулярная работа → «есть полная версия с планированием дня» | `personal_write` (структура) |

### Классификация на шаге 0

| Сигнал | Кто это | Путь |
|--------|---------|------|
| Нет ЦД, нет подписки | **Совсем новый** | Полный онбординг (шаги 1А→2А→3→...) |
| Есть ЦД, есть подписка БР | **Знакомый с МИМ** | Быстрый путь (шаги 1Б→2Б→3→...) |
| Есть ЦД + personal repos | **Опытный (T4)** | Не онбордим, работаем |
| Терминология в вопросе | **Опытный** | Не онбордим, работаем |

### Именование репозитория

Репо создаётся автоматически с именем **`DS-strategy`** (конвенция IWE). При переходе на T4 full — `setup.sh` обнаружит существующий репо и дополнит структуру, а не создаст с нуля. Пользователю имя не показывается — он видит «личная база знаний».

## Запрещённые слова (шаги 0-3)

GitHub, git, repo, fork, clone, commit, push, CLI, терминал, bash, npm, VS Code, IDE, DS-strategy, Downstream, Pack, MCP, Gateway, OAuth, API, webhook, CLAUDE.md, memory/, экзокортекс.

**Замены:** repo → «личная база знаний», commit → «сохранить», DS-strategy → «твоё пространство», OAuth → «подтверждение доступа», ЦД → «твой профиль».

## Три уровня шаблона (прогрессивный IWE)

| Уровень | Когда | Что создаётся | Возможности |
|---------|-------|---------------|-------------|
| **starter** | При регистрации (шаг 4) | inbox/, docs/, README.md | Заметки, поиск по знаниям, роли (Навигатор, Диагност) |
| **structured** | AI предлагает при 10+ записях | Тематические разделы | + Организация по темам, связи между записями |
| **full** | Ручной переход на Claude Code | FMT-exocortex-template | + ОРЗ-протоколы, memory, hooks, skills, day-open/close |

**Принцип:** Экзоскелет (DP.D.046) — усиливать, не перегружать. На старте минимум. AI предлагает следующий шаг, когда пользователь готов.

## Антипаттерны

| Антипаттерн | Правильно |
|-------------|-----------|
| Начинать с «подключи GitHub» | Сначала ценность, потом хранилище |
| Предлагать выбор имени репозитория | `DS-strategy` автоматически, без выбора |
| Объяснять архитектуру (тиры, Pack, DS) | Только если пользователь спросит |
| Требовать подписку на шаге 1 | Trial 30 дней, подписка позже |
| Давать 5+ вариантов действий | Один рекомендованный шаг |
| Использовать жаргон (экзокортекс, ОРЗ, ЦД) | Своими словами |

## Подписки (когда спрашивают)

Две подписки:
1. **Знания МИМ** — ~$10/мес на system-school.ru (фиксируется навсегда при автопродлении, позже ~$40/мес). Программы, руководства, персонализация, сообщество
2. **ИИ-помощник** — Claude Pro $20/мес (или GPT, Gemini). Сам диалог с ИИ

**Trial:** 30 дней полный доступ от /start. После — марафон + поиск знаний бесплатно.
