# [DP.SC.119] Рабочее пространство из браузера

## Обещание

**Кому:** Участник, который работает через браузер (claude.ai, ChatGPT, бот) и не использует VS Code / терминал.

**Зачем:** IWE не должно требовать технических навыков. Регистрация → GitHub OAuth → AI создаёт рабочее пространство. Пользователь не знает что такое git — и ему не нужно.

**Что получит:**
- **Рабочее пространство** — репозиторий создан автоматически, с минимальной структурой (inbox/, README.md)
- **Знания в контексте** — поиск по платформенным (L2) и своим (L4) знаниям через один Gateway URL
- **Запись и хранение** — заметки, выписки, рабочие документы сохраняются через AI, без ручного git
- **Прогрессивное усложнение** — по мере роста AI предлагает структурировать в Pack, подключить новые возможности

**Что НЕ входит в обещание:**
- Протоколы ОРЗ (day-open, close) — требуют Claude Code
- Skills с файловыми операциями — требуют доступ к FS
- Hooks, pre-commit проверки — требуют локальную среду
- Memory-система (auto-memory) — требует Claude Code

**Критерий приёмки:** От регистрации до первой записи через AI < 10 минут. Пользователь ни разу не видит git CLI.

## Пользовательский путь

> Участник регистрируется на платформе, подключает Gateway в браузерный AI-клиент, получает рабочее пространство и начинает работать — всё через диалог с AI.

| # | Шаг | Кто | Что происходит |
|---|-----|-----|----------------|
| 1 | Регистрация в Ory (Web App или бот) | Участник | Получает аккаунт платформы (SC.101) |
| 2 | Добавляет Gateway URL в claude.ai / ChatGPT | Участник | Одно действие: вставить `mcp.aisystant.com` как Connector |
| 3 | AI спрашивает: «Подключить GitHub?» → кнопка | Участник | GitHub OAuth — связать аккаунт. Пользователь не создаёт репо вручную |
| 4 | AI создаёт рабочий репозиторий из шаблона | Платформа | `create_repository` → GitHub API (из template repo `aisystant/iwe-starter`) |
| 5 | AI создаёт минимальную структуру | Платформа | `scaffold_iwe(level: starter)` → inbox/, README.md |
| 6 | Репо автоматически индексируется | Платформа | Webhook push → personal-knowledge-mcp → Neon |
| 7 | Участник работает: поиск, запись, вопросы AI | Участник | Через Gateway: search, write, get_instructions, роли |
| 8 | AI предлагает усложнение (при 10+ заметках) | Платформа | «Давай структурируем в Pack?» → `scaffold_pack` |
| 9 | (Опционально) Переход на Claude Code | Участник | Инструкция + полный FMT-шаблон. Разблокирует ОРЗ, memory, hooks |

## Три уровня шаблона (прогрессивный IWE)

| Уровень | Когда | Что создаётся | Возможности |
|---------|-------|---------------|-------------|
| **starter** | При регистрации | inbox/, docs/, README.md | Заметки, поиск по знаниям, роли (Навигатор, Диагност) |
| **pack** | AI предлагает при росте | Полная структура Pack по SPF | + Формализация знаний, methods, entities, distinctions |
| **full** | Ручной переход на Claude Code | FMT-exocortex-template | + ОРЗ-протоколы, memory, hooks, skills, day-open/close |

**Принцип:** Экзоскелет (DP.D.046) — усиливать, не перегружать. На старте минимум. AI предлагает следующий шаг, когда пользователь готов.
